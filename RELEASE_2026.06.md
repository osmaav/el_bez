# Релиз 2026.06 — Исправление ответов и хранение в Firestore

**Дата релиза:** 2 марта 2026 г.

**Версия:** 2026.06.0

---

## 🎉 Основные нововведения

### 📊 Поддержка любого количества ответов

Полная поддержка вопросов с различным количеством вариантов ответа:

- **2 ответа** — Да/Нет, Верно/Неверно
- **3 ответа** — Некоторые вопросы ПУЭ
- **4 ответа** — Стандартный формат (95% вопросов)
- **5+ ответов** — Редкие случаи

### 🔥 Хранение прогресса в Firestore

Прогресс обучения теперь сохраняется в облаке:

- **`user_states/{userId}.learningProgress`** — структура данных
- **Раздельное хранение** — для разделов 1256-19 и 1258-20
- **Сериализация** — `shuffledAnswers` сохраняется как JSON-строка
- **Автосинхронизация** — между устройствами для авторизованных пользователей

### ✅ Валидация данных

Автоматическая проверка корректности данных:

- **Проверка shuffledAnswers** — соответствие количеству ответов
- **Авто-перегенерация** — при несовпадении количества
- **Логирование** — для отладки проблем

### 📱 Улучшения UX

- **Кэширование вопросов** — мгновенное переключение разделов
- **Адаптивная навигация** — оптимизация для < 930px
- **Просмотр пароля** — кнопка с иконкой Eye/EyeOff
- **Запоминание отказа** — не показывать окно входа повторно

## 🐛 Исправления

### Критические

- 🐛 **Пустые варианты ответов** — исправлено для вопросов с 2-3 ответами
- 🐛 **Перезапись прогресса** — при переключении разделов
- 🐛 **Некорректное восстановление** — shuffledAnswers не соответствовал количеству ответов

### Важные

- 🐛 **Валидация страницы** — сброс на 1, если страница недоступна
- 🐛 **Кнопка "Сброс"** — очищает прогресс только текущего раздела
- 🐛 **Инициализация** — проверка `questions.length` перед загрузкой

### UX

- 🐛 **Отображение кнопки "Войти"** — иконка видна на всех экранах
- 🐛 **Подписи навигации** — скрыты на экранах < 930px
- 🐛 **Заголовок "Обучение"** — статический вместо динамического

## 🚀 Миграция на версию 2026.06

### Для пользователей

**Важно:** Прогресс обучения теперь синхронизируется через Firestore.

1. **Войдите в систему** — прогресс загрузится из облака
2. **Если нет аккаунта** — создайте учётную запись
3. **Прогресс из localStorage** — будет использован как fallback

### Для разработчиков

```bash
# Обновление зависимостей
npm install

# Запуск dev-сервера
npm run dev

# Сборка для продакшена
npm run build
```

**Настройка Firebase:**

1. Создайте проект в [Firebase Console](https://console.firebase.google.com/)
2. Включите Authentication (Email/Password)
3. Создайте Firestore Database
4. Конфигурация уже встроена в `src/lib/firebase.ts`
