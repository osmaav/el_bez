# 📋 Правила версионирования el-bez

**Версия документа:** 1.0  
**Дата:** 12 марта 2026

---

## 🎯 Формат версий

Используется формат **ГГГГ.ММ.ДД.XX** (YYYY.MM.DD.XX):

```
2026.03.12.00
│    │  │  │
│    │  │  └─ Номер релиза в день (00-99)
│    │  └──── День месяца (01-31)
│    └─────── Месяц (01-12)
└──────────── Год (4 цифры)
```

---

## 📊 Типы релизов

### .00 — Основной релиз (Main Release)

**Назначение:** Крупные релизы с новой функциональностью

**Пример:** `2026.03.12.00`

**Когда использовать:**
- ✅ Новая функциональность
- ✅ Крупные изменения архитектуры
- ✅ Релиз после слияния больших веток
- ✅ Плановые релизы

**Ветки:**
- `feature/*` → `test` → `master`

---

### .01–.09 — Исправления (Bug Fixes)

**Назначение:** Исправления ошибок и багфиксы

**Примеры:**
- `2026.03.12.01` — первое исправление
- `2026.03.12.02` — второе исправление
- `2026.03.12.09` — девятое исправление

**Когда использовать:**
- 🐛 Критические баги
- 🐛 Исправления тестов
- 🐛 Хотфиксы production
- 🐛 Исправления документации

**Ветки:**
- `hotfix/*` → `master`

---

### .10–.99 — Новый функционал (Features)

**Назначение:** Добавление новых функций и улучшений

**Примеры:**
- `2026.03.12.10` — первое крупное изменение
- `2026.03.12.11` — второе крупное изменение
- `2026.03.12.99` — 90-е изменение в день

**Когда использовать:**
- ✨ Новые функции
- ✨ Улучшения существующих функций
- ✨ Рефакторинг
- ✨ Оптимизация

**Ветки:**
- `feature/*` → `test` → `master`

## 🔄 Процесс релиза

### Основной релиз (.00)

```bash
# 1. Создание ветки
git checkout -b release/2026.03.12.00

# 2. Внесение изменений
# ... работа над функциями ...

# 3. Тестирование
npm run test

# 4. Слияние с test
git checkout test
git merge release/2026.03.12.00

# 5. Финальное тестирование
npm run test

# 6. Слияние с master
git checkout master
git pull origin master
git merge test --no-ff -m "merge: Release 2026.03.12.00"

# 7. Публикация
git push origin master
git tag 2026.03.12.00
git push origin 2026.03.12.00
```

### Исправление (.01–.09)

```bash
# 1. Создание ветки hotfix
git checkout -b hotfix/2026.03.12.01

# 2. Исправление бага
# ... исправление ...

# 3. Тестирование
npm run test

# 4. Слияние с master
git checkout master
git merge hotfix/2026.03.12.01 --no-ff -m "merge: Hotfix 2026.03.12.01"

# 5. Публикация
git push origin master
git tag 2026.03.12.01
git push origin 2026.03.12.01
```

### Новый функционал (.10–.99)

```bash
# 1. Создание ветки feature
git checkout -b feature/2026.03.12.10

# 2. Разработка функции
# ... разработка ...

# 3. Тестирование
npm run test

# 4. Слияние с test
git checkout test
git merge feature/2026.03.12.10

# 5. Слияние с master
git checkout master
git merge test --no-ff -m "merge: Feature 2026.03.12.10"

# 6. Публикация
git push origin master
git tag 2026.03.12.10
git push origin 2026.03.12.10
```

## 🏷️ Теги Git

### Создание тега

```bash
# Аннотированный тег
git tag -a 2026.03.12.00 -m "Release 2026.03.12.00 - Tests Integration"

# Лёгкий тег
git tag 2026.03.12.01
```

### Публикация тега

```bash
# Один тег
git push origin 2026.03.12.00

# Все теги
git push origin --tags
```

### Просмотр тегов

```bash
# Все теги
git tag -l

# Теги с описанием
git tag -n

# Конкретный тег
git show 2026.03.12.00
```

## 📅 Ежедневные релизы

**Рекомендация:** Создавать основной релиз `.00` ежедневно в конце рабочего дня.

**Пример расписания:**
- `2026.03.12.00` — основной релиз за 12 марта
- `2026.03.13.00` — основной релиз за 13 марта
- `2026.03.14.00` — основной релиз за 14 марта

**Преимущества:**
- ✅ Свежая версия всегда доступна
- ✅ Легко откатиться на вчерашнюю версию
- ✅ Понятная хронология изменений
- ✅ Прозрачная история релизов
