# ✅ Исправление: Применение фильтров по кнопке

## Проблема
При переключении переключателей "Исключить известные" или "Исключить слабые" фильтры применялись сразу же, без нажатия кнопки "Применить".

**Ожидаемое поведение:**
1. Пользователь выбирает фильтры (переключатели)
2. Фильтры НЕ применяются сразу
3. Пользователь нажимает кнопку "Применить"
4. Фильтры применяются

**Фактическое поведение (до исправления):**
1. Пользователь переключает переключатель
2. Фильтры применяются сразу ❌

## Причина
В коде были два места которые вызывали `applyFilters()` автоматически:

1. `useEffect` который отслеживал изменение переключателей:
```typescript
React.useEffect(() => {
  applyFilters();
}, [excludeKnown, excludeWeak]);
```

2. `setTimeout(applyFilters, 0)` в обработчиках `onCheckedChange`:
```typescript
onCheckedChange={(checked) => {
  setExcludeKnown(checked);
  setTimeout(applyFilters, 0); // ❌ Применяет сразу
}}
```

## Решение

### 1. Удалён `useEffect` для автоматического применения
```typescript
// ❌ БЫЛО
React.useEffect(() => {
  applyFilters();
}, [excludeKnown, excludeWeak]);

// ✅ СТАЛО
// useEffect удалён - фильтры применяются только по кнопке
```

### 2. Убран `setTimeout` из обработчиков
```typescript
// ❌ БЫЛО
onCheckedChange={(checked) => {
  setExcludeKnown(checked);
  setTimeout(applyFilters, 0);
}}

// ✅ СТАЛО
onCheckedChange={(checked) => {
  setExcludeKnown(checked);
  // Применение только по кнопке "Применить"
}}
```

### 3. Кнопка "Применить" работает корректно
```typescript
<Button
  variant="default"
  size="sm"
  onClick={applyFilters} // ✅ Вызывает applyFilters только по клику
  className="flex-1 bg-blue-600 hover:bg-blue-700"
>
  Применить
</Button>
```
