**Версия:** 2026.03.25.00

## 📋 Обзор изменений

Этот релиз посвящён улучшению работы приложения на мобильных устройствах с экранами шириной 390px (iPhone и аналогичные устройства). Основное изменение — отказ от JavaScript-определения мобильного устройства в пользу стандартных CSS-брейкпоинтов Tailwind.

---

## ✨ Что нового

### 1. **Удалён хук `use-mobile.ts`**
- ❌ Удалён файл `src/hooks/use-mobile.ts`
- ❌ Удалена функция `useIsMobile()`
- ❌ Удалена константа `MOBILE_BREAKPOINT = 768`

**Причина:** Хук дублировал функциональность CSS-медиазапросов и создавал проблемы с определением мобильных устройств на экранах 390px.

### 2. **Обновлён компонент `Sidebar`**
- ✅ Мобильное/десктопное отображение теперь определяется через CSS-классы:
  - `<Sheet>` для мобильных: добавлен класс `md:hidden`
  - `<div>` для десктопа: `hidden md:block`
- ✅ Убрано условное ветвление `if (isMobile)`
- ✅ Оба варианта рендерятся одновременно, CSS скрывает ненужный

**Пример:**
```tsx
// Было
if (isMobile) {
  return <Sheet>...</Sheet>
}
return <div>...</div>

// Стало
return (
  <>
    <Sheet className="md:hidden">...</Sheet>
    <div className="hidden md:block">...</div>
  </>
)
```

### 3. **Обновлён компонент `SidebarMenuButton`**
- ✅ Удалено использование `isMobile` из контекста
- ✅ Tooltip теперь скрывается только по условию `state !== "collapsed"`
- ✅ Упрощена логика показа подсказок

**До:**
```tsx
const { isMobile, state } = useSidebar()
<TooltipContent hidden={state !== "collapsed" || isMobile} />
```

**После:**
```tsx
const { state } = useSidebar()
<TooltipContent hidden={state !== "collapsed"} />
```

## 🔧 Технические детали

### Стандартные брейкпоинты Tailwind

Теперь используются стандартные значения Tailwind CSS:

| Префикс | Мин. ширина | Устройства |
|---------|-------------|------------|
| `sm:` | 640px | Мобильные (портрет) |
| `md:` | 768px | Планшеты |
| `lg:` | 1024px | Ноутбуки |
| `xl:` | 1280px | Десктопы |
| `2xl:` | 1536px | Большие экраны |

### Почему это работает лучше

**Проблема до:**
- `useIsMobile()` использовал `window.matchMedia("(max-width: 767px)")`
- На iPhone 390px хук возвращал `true`
- Классы `sm:` не работали, так как 390px < 640px

**Решение после:**
- CSS `md:` работает нативно на всех устройствах
- 390px < 768px → применяется мобильная версия
- 640px ≤ 768px → работают классы `sm:`
